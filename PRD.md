# SmartFit 产品需求文档 (PRD)

## 一、项目概述

### 1.1 产品定位
SmartFit 是一款智能穿搭助手 Web 应用，为用户提供基于身材数据、色彩季型、天气和场合的个性化穿搭建议。

### 1.2 目标用户
- 对穿搭有需求但缺乏灵感的年轻人
- 希望优化衣橱管理的用户
- 网购服装前想预览效果的消费者

### 1.3 核心价值
- **智能推荐**：基于多维度数据的穿搭方案
- **色彩诊断**：帮助用户找到适合自己的颜色
- **衣橱管理**：整理现有衣物，提高利用率
- **虚拟试衣**：降低网购风险
- **购物助手**：精准推荐购买链接

### 3.3 色彩测试 ⭐ 新功能
**页面路径**: `/color-test`

**功能说明**:
根据用户的肤色、发色、瞳孔颜色等特征，判断用户属于哪个色彩季型（春/夏/秋/冬），推荐适合的颜色。

**测试流程**:
1. **问题 1**: 你的肤色偏向？
   - 选项：暖调（偏黄）/ 冷调（偏粉）/ 中性
   
2. **问题 2**: 你的自然发色是？
   - 选项：深黑色 / 棕黑色 / 深棕色 / 浅棕色 / 金棕色
   
3. **问题 3**: 你佩戴金色饰品还是银色饰品更显气色？
   - 选项：金色 / 银色 / 都可以
   
4. **问题 4**: 你的眼睛颜色？
   - 选项：深黑褐色 / 浅褐色 / 棕色 / 其他

5. **问题 5**: 穿纯白色 vs 米白色，哪个让你看起来更有精神？
   - 选项：纯白色 / 米白色 / 差不多

**诊断结果**:
- **春季型（暖调+明亮）**：适合暖色调、明亮的颜色
  - 推荐：珊瑚色、桃粉色、浅黄、浅绿、驼色
  - 避免：纯黑、纯白、深灰
  
- **夏季型（冷调+柔和）**：适合冷色调、柔和的颜色
  - 推荐：浅蓝、薰衣草紫、浅粉、灰粉、烟灰色
  - 避免：橙色、金色、大红色
  
- **秋季型（暖调+沉稳）**：适合暖色调、深沉的颜色
  - 推荐：深棕、砖红、墨绿、芥末黄、驼色
  - 避免：粉色、浅蓝、荧光色
  
- **冬季型（冷调+鲜明）**：适合冷色调、鲜艳或极深的颜色
  - 推荐：纯白、纯黑、宝蓝、酒红、亮粉
  - 避免：橙色、驼色、浅黄

**数据存储**:
```typescript
{
  seasonType: 'spring' | 'summer' | 'autumn' | 'winter',
  recommendColors: string[],
  avoidColors: string[]
}
```

### 3.4 我的衣橱 ⭐ 新功能
**页面路径**: `/wardrobe`

**功能说明**:
用户可以管理自己现有的衣物，系统会根据衣橱内容推荐穿搭组合。

**核心功能**:
1. **衣物列表展示**
   - 分类：上装/下装/外套/鞋子/配饰
   - 网格布局展示
   - 显示：图片、名称、颜色、购买渠道

2. **添加衣物**
   - 上传衣物图片（或选择默认图标）
   - 填写基本信息
     - 类型：上装/下装/外套/鞋子/配饰
     - 颜色：从色板选择
     - 风格：休闲/正式/运动/街头
     - 名称：自定义
     - 购买链接（可选）

3. **衣物详情**
   - 查看图片
   - 编辑/删除

**数据结构**:
```typescript
interface ClothingItem {
  id: string
  image: string
  name: string
  type: 'top' | 'bottom' | 'outerwear' | 'shoes' | 'accessory'
  color: string
  style: 'casual' | 'formal' | 'sport' | 'street'
  purchaseLink?: string
  addedAt: Date
}
```

**交互设计**:
- 顶部分类标签切换
- 右下角 FAB 按钮 "+" 添加衣物
- 点击卡片查看详情
- Mock 数据：预置 10-15 件常见衣物

### 3.5 每日推荐
**页面路径**: `/recommend`

**功能说明**:
基于天气、场合、用户衣橱，生成今日穿搭建议。

**输入条件**:
- 天气信息（Mock：晴天 22°C）
- 场合：通勤/约会/运动/街头
- 用户色彩季型
- 用户衣橱（如果有）

**推荐逻辑**:
1. 根据温度选择厚度
2. 根据场合选择风格
3. 根据色彩季型选择颜色搭配
4. 优先推荐用户衣橱中的衣物
5. 补充推荐购买单品

**展示形式**:
- 方案 A：整体穿搭效果图（Mock 图片）
- 方案 B：单品组合（从衣橱 + 推荐商品）
- 方案 C：备选方案

## 四、技术方案

### 4.1 技术栈
- **框架**: Next.js 16 (App Router)
- **UI**: React 19 + TailwindCSS 4
- **动画**: Framer Motion
- **图标**: Lucide React
- **状态管理**: React Hooks (useState, useContext)
- **数据存储**: LocalStorage (本地持久化)

### 4.2 Mock 数据策略
由于当前阶段不接入真实 API，使用以下 Mock 方案：

1. **用户数据**: 存储在 LocalStorage
2. **天气数据**: 固定返回 `{ temperature: 22, condition: 'sunny' }`
3. **穿搭图片**: 使用网络图片或 `/public` 静态资源
4. **商品数据**: 预置 Mock 数据数组
5. **衣橱数据**: 预置示例衣物 + 用户添加的存储在 LocalStorage

### 4.3 数据流程
```
用户输入 → LocalStorage 存储 → Mock 数据处理 → 页面展示
```

## 五、设计规范

### 5.1 配色方案（极简黑白灰）
```css
--primary-black: #000000      /* 主色：文字、按钮 */
--secondary-gray: #666666     /* 次要文字 */
--border-gray: #e5e5e5        /* 边框 */
--bg-white: #ffffff           /* 背景白色 */
--bg-gray: #f8f8f8            /* 浅灰背景 */
--accent-black: #1a1a1a       /* 强调色 */
```

**禁止使用**:
- ❌ 渐变色
- ❌ 鲜艳的彩色背景
- ❌ 过于花哨的 AI 风格界面
- ❌ 荧光色、霓虹色

**允许使用**（作为点缀）:
- ✅ 色彩测试页面的色块（功能必需）
- ✅ 衣物图片本身的颜色
- ✅ 极少量的品牌色（logo）

### 5.2 排版规范
- **字体**: Inter (英文), 系统默认 (中文)
- **字号**:
  - 标题: 24px / 20px
  - 正文: 14px
  - 辅助文字: 12px / 11px
- **行距**: 1.5
- **对齐**: 左对齐为主
- **留白**: 充足的内边距和外边距

### 5.3 组件风格
- **按钮**: 直角边框、1px 黑色描边、无圆角
- **输入框**: 极简风格、底部下划线或完整边框
- **卡片**: 白色背景、浅灰边框、轻微阴影
- **图标**: 线性图标（Lucide）

### 5.4 布局原则
- 移动端优先（最大宽度 430px）
- 居中对齐
- 网格布局（Grid）
- 一致的间距系统（8px 基准）

## 八、风险与应对

### 8.1 技术风险
- **风险**: Mock 数据与真实 API 差异大
- **应对**: 设计时预留 API 接口，使用统一的数据层

### 8.2 用户体验风险
- **风险**: 虚拟试衣效果不够真实
- **应对**: 使用高质量示例图，添加说明文字

### 8.3 项目范围风险
- **风险**: 功能过多导致无法按时完成
- **应对**: 分阶段开发，优先保证核心流程
