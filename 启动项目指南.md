# 🚀 SmartFit 项目启动指南

## 📋 快速开始

### 1. 打开终端/命令行

**Windows:**
- 按 `Win + R`，输入 `cmd` 或 `powershell`，回车
- 或者在项目文件夹中，按住 `Shift + 右键`，选择"在此处打开 PowerShell 窗口"

**Mac/Linux:**
- 按 `Cmd + Space` (Mac) 或 `Ctrl + Alt + T` (Linux)，打开终端

### 2. 进入项目目录

```bash
cd C:\Users\34023\Desktop\yzh\smart-fit
```

### 3. 安装依赖（首次运行需要）

```bash
npm install
```

**说明：** 如果 `node_modules` 文件夹已存在，可以跳过此步骤。

### 4. 启动开发服务器

```bash
npm run dev
```

### 5. 打开浏览器

启动成功后，你会看到类似这样的输出：

```
  ▲ Next.js 16.0.3
  - Local:        http://localhost:3000
  - Network:      http://192.168.x.x:3000
```

在浏览器中访问：**http://localhost:3000**

## 📝 常用命令

### 开发模式
```bash
npm run dev
```
- 启动开发服务器
- 支持热重载（代码修改后自动刷新）
- 默认端口：3000

### 构建生产版本
```bash
npm run build
```
- 构建优化后的生产版本
- 生成 `.next` 文件夹

### 启动生产服务器
```bash
npm start
```
- 运行构建后的生产版本
- 需要先执行 `npm run build`

### 代码检查
```bash
npm run lint
```
- 运行 ESLint 检查代码质量

## 🛠️ 故障排除

### 问题 1: 端口被占用

**错误信息：** `Port 3000 is already in use`

**解决方法：**
1. 关闭占用 3000 端口的程序
2. 或者使用其他端口：
   ```bash
   npm run dev -- -p 3001
   ```

### 问题 2: 依赖安装失败

**错误信息：** `npm ERR!` 相关错误

**解决方法：**
1. 清除缓存：
   ```bash
   npm cache clean --force
   ```
2. 删除 `node_modules` 和 `package-lock.json`：
   ```bash
   rm -rf node_modules package-lock.json
   ```
   Windows:
   ```powershell
   Remove-Item -Recurse -Force node_modules, package-lock.json
   ```
3. 重新安装：
   ```bash
   npm install
   ```

### 问题 3: Node.js 版本不兼容

**要求：** Node.js 18.x 或更高版本

**检查版本：**
```bash
node -v
```

**升级 Node.js：**
- 访问 https://nodejs.org/ 下载最新版本
- 或使用 nvm (Node Version Manager)
